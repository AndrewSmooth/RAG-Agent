# Table: department
## Notes
- Каждый отдел должен быть привязан к одному бизнес-подразделению
- Поле `code` должно быть уникальным в пределах одного бизнес-подразделения
- Иерархия отделов строится через поле `parent_dept_id`
- Разница между `headcount_plan` и `headcount_current` используется для анализа кадровой ситуации
- При реорганизации отделов старые записи не удаляются, а получают статус 'reorganized'
- Бюджет отдела является частью бюджета бизнес-подразделения
- Все кадровые операции с сотрудниками привязываются к конкретному отделу
