# Table: maintenance_record
## Columns
- `id`: уникальный идентификатор записи о техническом обслуживании (первичный ключ)
- `machine_id`: ссылка на оборудование (внешний ключ)
- `schedule_id`: ссылка на график ТО (внешний ключ, если применимо)
- `work_order_id`: ссылка на наряд-заказ (внешний ключ)
- `maintenance_type`: тип обслуживания (preventive, predictive, corrective, emergency)
- `start_date`: дата начала работ
- `end_date`: дата окончания работ
- `downtime_hours`: время простоя оборудования
- `labor_hours`: затраченные трудозатраты
- `technician_id`: идентификатор техника
- `supervisor_id`: идентификатор руководителя работ
- `work_description`: описание выполненных работ
- `findings`: выявленные неисправности и наблюдения
- `actions_taken`: выполненные действия по устранению неисправностей
- `parts_replaced`: замененные запчасти (в формате JSON)
- `tools_used`: использованные инструменты (в формате JSON)
- `safety_violations`: выявленные нарушения безопасности (в формате JSON)
- `quality_check_required`: требуется ли проверка качества после работ
- `quality_check_passed`: пройдена ли проверка качества
- `rework_required`: требуется ли переделка работ
- `actual_cost`: фактическая стоимость выполнения
- `cost_currency`: валюта стоимости
- `status`: статус записи (completed, in_progress, cancelled)
- `created_at`: дата и время создания записи
- `updated_at`: дата и время последнего обновления записи
- `created_by`: идентификатор пользователя, создавшего запись
- `updated_by`: идентификатор пользователя, последним обновившего запись
- `is_active`: флаг активности записи (true/false)
- `version`: версия записи для контроля параллельных изменений
