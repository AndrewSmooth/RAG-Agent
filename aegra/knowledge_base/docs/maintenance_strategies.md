# Стратегии технического обслуживания

## Определение
Техническое обслуживание — это комплекс мероприятий, направленных на поддержание оборудования в работоспособном состоянии и предотвращение его отказов. Выбор правильной стратегии технического обслуживания критически важен для обеспечения надежности производства, минимизации простоев и оптимизации затрат.

## Типы стратегий технического обслуживания

### 1. Реактивное обслуживание (Reactive Maintenance)
- Обслуживание после отказа оборудования
- "Ломалось — чини"
- Минимальные затраты на планирование
- Высокие затраты на простой и аварийный ремонт
- Подходит для некритичного оборудования

### 2. Планово-предупредительное обслуживание (Preventive Maintenance, PM)
- Регулярное обслуживание по календарному графику или наработке
- Основано на среднем времени между отказами (MTBF)
- Снижает вероятность отказов
- Может приводить к излишнему обслуживанию
- Использует данные из таблицы [`maintenance_schedule`](rag/knowledge_base/docs/maintenance_schedule.md:1)

### 3. Прогнозирующее обслуживание (Predictive Maintenance, PdM)
- Обслуживание на основе состояния оборудования
- Использует мониторинг параметров (вибрация, температура, шум)
- Определяет начало износа до отказа
- Оптимизирует интервалы обслуживания
- Требует инвестиций в датчики и системы мониторинга

### 4. Условно-предупредительное обслуживание (Condition-Based Maintenance, CBM)
- Подмножество прогнозирующего обслуживания
- Обслуживание при достижении пороговых значений параметров
- Более простая реализация по сравнению с PdM
- Требует установки датчиков

### 5. Обслуживание по надежности (Reliability-Centered Maintenance, RCM)
- Систематический подход к определению оптимальной стратегии для каждого оборудования
- Анализирует функции, отказы и последствия отказов
- Выбирает наиболее эффективную стратегию для каждого типа отказа
- Оптимизирует затраты и надежность

### 6. Обслуживание, ориентированное на функции (Function-Oriented Maintenance)
- Фокус на поддержании функций оборудования, а не на самом оборудовании
- Рассматривает систему в целом
- Позволяет использовать альтернативные решения при отказе

## Критерии выбора стратегии
При выборе стратегии технического обслуживания учитываются:

### 1. Критичность оборудования
- Влияние отказа на производство
- Влияние на безопасность
- Влияние на окружающую среду
- Финансовые последствия простоя

### 2. Характер отказов
- Случайные или закономерные
- Постепенные или внезапные
- Возрастные или не зависящие от возраста

### 3. Затраты
- Стоимость обслуживания
- Стоимость простоя
- Стоимость запасных частей
- Стоимость персонала

### 4. Доступность данных
- Наличие истории отказов
- Возможность мониторинга состояния
- Наличие технической документации

## Применение в системе
Наша ERP-система поддерживает все основные стратегии технического обслуживания:

### 1. Планирование обслуживания
- В таблице [`maintenance_schedule`](rag/knowledge_base/docs/maintenance_schedule.md:1) определены типы обслуживания для каждого оборудования
- Система автоматически генерирует наряды-заказы на обслуживание
- Учитывает критичность оборудования из таблицы [`machine`](rag/knowledge_base/docs/machine.md:1)

### 2. Мониторинг состояния
- Интеграция с системами SCADA и IoT для сбора данных о состоянии оборудования
- Хранение данных в таблице `machine_sensors` (предполагаемая)
- Анализ данных для прогнозирования отказов

### 3. Управление нарядами-заказами
- Все работы регистрируются в таблице [`maintenance_record`](rag/knowledge_base/docs/maintenance_record.md:1)
- Отслеживаются трудозатраты, затраты, время простоя
- Оценивается эффективность выполненных работ

### 4. Анализ и оптимизация
- Система рассчитывает показатели эффективности обслуживания:
  - MTBF (Mean Time Between Failures)
  - MTTR (Mean Time To Repair)
  - Коэффициент готовности
  - Затраты на обслуживание на единицу продукции
- Генерируются отчеты по эффективности стратегий обслуживания
- Предлагаются рекомендации по оптимизации

## Показатели эффективности технического обслуживания
### 1. Коэффициент готовности (Availability)
```
Коэффициент готовности = (Время работы / (Время работы + Время простоя)) × 100%
```

### 2. Среднее время между отказами (MTBF)
```
MTBF = Общее время работы / Количество отказов
```

### 3. Среднее время восстановления (MTTR)
```
MTTR = Общее время простоя / Количество отказов
```

### 4. Коэффициент эффективности обслуживания
```
Коэффициент = (Плановые работы / (Плановые работы + Аварийные работы)) × 100%
```

Высокое значение (более 80%) указывает на эффективное планирование.

### 5. Затраты на техническое обслуживание
- Затраты на обслуживание на единицу продукции
- Затраты на обслуживание на единицу времени
- Соотношение плановых и аварийных затрат

## Непрерывное улучшение
Система поддерживает цикл улучшения обслуживания:
1. Анализ текущих показателей
2. Выявление проблемных областей
3. Разработка и реализация улучшений
4. Оценка эффективности изменений
5. Стандартизация успешных решений

Также поддерживается методология TPM (Total Productive Maintenance) для вовлечения всего персонала в поддержание оборудования.
