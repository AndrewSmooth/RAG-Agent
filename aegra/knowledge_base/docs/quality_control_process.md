# Процесс контроля качества

## Определение
Контроль качества — это систематический процесс проверки и оценки продукции и производственных процессов для обеспечения соответствия установленным стандартам и требованиям. Цель контроля качества — предотвратить выпуск бракованной продукции и обеспечить стабильность производственных процессов.

## Принципы контроля качества
- **Предотвращение, а не обнаружение**: фокус на предотвращении дефектов, а не на их выявлении
- **Ответственность всех**: каждый сотрудник несет ответственность за качество
- **Процессный подход**: качество определяется качеством процессов
- **Основанный на фактах**: принятие решений на основе данных и анализа
- **Постоянное улучшение**: непрерывная работа по улучшению качества

## Этапы контроля качества
### 1. Планирование контроля качества
- Разработка плана контроля качества (Quality Control Plan, QCP)
- Определение контрольных точек в производственном процессе
- Установление методов и частоты контроля
- Определение критериев приемки и отбраковки

### 2. Входной контроль (Incoming Inspection)
- Проверка поступающих материалов и компонентов
- Проверка соответствия спецификациям
- Проверка сертификатов качества от поставщиков
- Отбор проб и лабораторные испытания

### 3. Производственный контроль (In-Process Inspection)
- Проверка продукции на ключевых этапах производства
- Проверка параметров процесса (температура, давление, скорость и т.д.)
- Проверка калибровки оборудования
- Аудит производственных процессов

### 4. Окончательный контроль (Final Inspection)
- Проверка готовой продукции перед упаковкой
- Проверка соответствия требованиям заказа
- Проверка упаковки и маркировки
- Выборочная проверка или 100% контроль в зависимости от риска

### 5. Контроль перед поставкой (Pre-Shipment Inspection)
- Финальная проверка перед отгрузкой
- Проверка комплектности
- Проверка условий хранения и транспортировки
- Подготовка сертификатов качества

## Методы контроля качества
### 1. Выборочный контроль
- Контроль части партии по статистическим методам
- Использует стандарты (например, ISO 2859, ГОСТ Р 50779.71)
- Определяет объем выборки и критерии приемки

### 2. 100% контроль
- Проверка каждой единицы продукции
- Применяется для критичных компонентов или при высоком риске
- Часто автоматизируется

### 3. Автоматический контроль
- Использование автоматизированных систем (камеры, датчики, роботы)
- Непрерывный мониторинг параметров
- Мгновенное обнаружение отклонений

### 4. Статистический контроль процессов (SPC)
- Использование контрольных карт для мониторинга процессов
- Анализ вариабельности процесса
- Прогнозирование отклонений

## Показатели качества
### 1. Уровень брака
```
Уровень брака = (Количество бракованных единиц / Общее количество произведенных единиц) × 100%
```

### 2. Коэффициент выхода годной продукции (First Pass Yield)
```
FPY = (Количество единиц, прошедших контроль с первого раза / Общее количество единиц) × 100%
```

### 3. Индекс дефектов на миллион возможностей (DPMO)
```
DPMO = (Количество дефектов / (Общее количество единиц × Количество возможностей для дефекта)) × 1,000,000
```

### 4. Удовлетворенность клиентов
- Количество жалоб клиентов
- Уровень возвратов
- Рейтинги удовлетворенности

## Применение в системе
Наша ERP-система интегрирует контроль качества во все производственные процессы:

### 1. Планирование контроля
- В таблице [`routing`](rag/knowledge_base/docs/routing.md:1) определены контрольные точки качества
- В таблице [`operation`](rag/knowledge_base/docs/operation.md:1) указаны требования к контролю
- В таблице [`bom`](rag/knowledge_base/docs/bom.md:1) указаны требования к качеству материалов

### 2. Регистрация результатов контроля
- Результаты входного контроля регистрируются в таблице `incoming_inspection` (предполагаемая)
- Результаты производственного контроля — в таблице [`work_order`](rag/knowledge_base/docs/work_order.md:1) (поле `quality_checks_passed`)
- Результаты окончательного контроля — в таблице [`production_order`](rag/knowledge_base/docs/production_order.md:1) (поля `scrap_quantity`, `produced_quantity`)

### 3. Управление несоответствиями
- При выявлении дефекта создается запись в таблице `non_conformance` (предполагаемая)
- Определяется причина дефекта (материал, процесс, персонал, оборудование)
- Назначается ответственный за устранение
- Отслеживается выполнение корректирующих действий

### 4. Анализ качества
- Система автоматически рассчитывает показатели качества
- Генерируются отчеты по качеству по продуктам, линиям, сменам
- Выявляются тренды и проблемы с качеством
- Предлагаются рекомендации по улучшению

## Непрерывное улучшение качества
Система поддерживает цикл PDCA (Plan-Do-Check-Act):
- **Plan**: планирование улучшений на основе анализа данных
- **Do**: реализация улучшений
- **Check**: проверка эффективности улучшений
- **Act**: стандартизация успешных решений

Также поддерживается методология шести сигм (Six Sigma) для снижения вариабельности процессов и повышения качества.
