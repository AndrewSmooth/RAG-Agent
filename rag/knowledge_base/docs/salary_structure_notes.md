# Table: salary_structure
## Notes
- Структура оклада определяет базовые значения для расчета зарплаты сотрудников
- Каждая запись действует в определенный период времени (effective_date - expiration_date)
- Региональный коэффициент применяется в зависимости от местоположения сотрудника
- Окончательный оклад рассчитывается по формуле: base_salary * region_factor * experience_factor * performance_factor
- Диапазон оклада (range_min - range_max) используется для контроля при назначении зарплаты
- Midpoint представляет собой целевое значение оклада для данного грейда
- При индексации окладов создаются новые записи с новыми effective_date
- Старые записи не удаляются, а получают expiration_date
